# research-loom —— 文献→设计 工作方法

> 本篇是**本项目自身的研究→设计工作方法**，非 autoharness 产品架构（产品架构见 [workflow.md](workflow.md)）。
> 操作指南由 `research-loom` skill 承载；内容产物落在 `docs/research-loom/`，其字段契约见该目录索引。

## 原则

**把"读文献到出设计"做成一套原子稳定、视图可弃、全程可溯的分层结构，使想法随阅读演化而不必重构。结构而非次序：层间只有 provenance 依赖，没有时间先后。**
病根在于把思考存成一棵不断生长的树：树逼人在想清前先定层级，理解一变层级即错位，于是重构、且旧结论被覆盖而遗忘。
解法是拆开**原子**（小而自洽、有稳定标识、几乎不动）与**视图**（矩阵、脊柱图、思维导图等廉价可弃的投影）：原子稳定则"想法变了"只意味着加原子、重画视图，而非重构。

## 五层（结构，非时间顺序）

五者是内容的**结构层，不是先后步骤**。按真实节奏跨层穿梭（调研一批就先搭设计、随后再调研、随时回改 idea），任意顺序、可交错。层间箭头是 **provenance 依赖**（一块卡建立在什么之上），不是时间次序：

```
源卡 ◄─引用─ idea ◄─建立在─ 设计 ◄─作用于─ 决策
sources/      ideas/       design/    decisions/
  ▲
  └─横向对比─ 方向 synthesis/
```

子目录均在 `docs/research-loom/` 下。维护的是**归层、原子、格式、链接**，不是次序。

- **来源**（`sources/`，内分 `papers/` `github/` `blogs/` …）—— 每篇一张**文献笔记**：核心论点、做法、优缺点。**按类型自动归类**，开放集，无匹配则新建一类。事实层，不掺判断。
- **方向**（`synthesis/`）—— 把来源横向聚类对比，每方向一张 MOC + 概念矩阵 + "我的评判"。按概念组织，不按作者。
- **idea**（`ideas/`）—— 把认可的思路与新念头提为**永久笔记**：用自己的话、原子化、引用所凭来源。不是摘要，是你的主张。
- **设计**（`design/`）—— 把 idea 织成设计脊柱，每元素连 provenance（→ idea → 源卡）。
- **决策**（`decisions/`）—— 对设计上有争议的点跑决策模型（驱动力→选项→权衡→取舍），坍缩成 ADR，作用回设计。

## 不变量

1. **三种卡分家。** 文献笔记（论文说）、永久笔记（我说）、设计（组装）是三类不同的原子；文献↔永久分家，才使念头能自由重组进设计而不必重构。
2. **provenance 可溯。** 任何设计元素经永久笔记可一路追回支撑它的文献，"链接到各论文"由此自动成立。
3. **非线性维护，无时间顺序。** 五层任意可增改、跨层交错（调研与设计反复穿插，调研 N 篇即可先设计、再调研再回改）。层间是 provenance 依赖（决策作用于设计、设计建立在 idea、idea 引用源卡），不是先后步骤；可自顶向下（先搭设计再补 idea/源）也可自底向上。skill 只维护归层、原子、格式、链接。
4. **决策只增不删（append-only）。** 旧决策被取代而非抹除，留下"为何如此、后又为何改"的轨迹，对抗遗忘。
5. **视图由原子投影，不单独维护。** 概念矩阵、脊柱图、思维导图皆从原子的结构化元数据生成，可弃可重建；唯原子长期供养。
6. **载体中立、可升级。** 原子为纯文本加结构化元数据，**卡间用相对 Markdown 链接互连**（GitHub 可渲染、笔记工具可反链、可被链接校验器核查）；`[[ ]]` 不作链接机制，以免与散文中的示意性 `[[ ]]` 混淆。同一份原子既可在纯仓库中检索，亦可被笔记工具渲染为自动视图。

## 与产品设计树的过渡

`research-loom/design/` 是将来唯一的设计家；现有 `docs/design/`（含本篇）在过渡期冻结为 legacy，新设计落 `research-loom/design/`，过渡期以它为准，日后整体迁移。此举避免两个真相源。
